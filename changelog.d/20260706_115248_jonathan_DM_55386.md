### New features

- Added an external link-checking service with a submit-and-poll API. `POST /ook/linkcheck/checks` accepts a documentation build's external URLs (an LSST the Docs project slug, a default-branch flag, and a list of URLs with the page paths they occur on). URLs are canonicalized (fragments stripped) and partitioned: URLs with a fresh cached result and unsupported (non-http(s)) URLs resolve immediately, while the rest are checked asynchronously. The endpoint returns 202 with a `Location` header for polling. `GET /ook/linkcheck/checks/{id}` reports the check's processing status (pending/in_progress/complete), summary counts by URL status, and per-URL results (status, HTTP status code, redirect location). Only default-branch submissions replace a project's recorded URL occurrences; PR-build submissions still receive full results. Link health is tracked per canonical URL across projects with a failing-to-broken retry ladder, so a previously-OK link is only declared broken after repeated failures over a configurable window.
- New configuration settings for link checking: `OOK_LINKCHECK_REQUEST_TIMEOUT`, `OOK_LINKCHECK_MAX_CONCURRENCY`, `OOK_LINKCHECK_HOST_INTERVAL`, `OOK_LINKCHECK_FRESHNESS_TTL`, and `OOK_LINKCHECK_MAX_URLS_PER_CHECK`. The failing-to-broken retry ladder is tunable through `OOK_LINKCHECK_BROKEN_THRESHOLD` (default 48h), `OOK_LINKCHECK_BROKEN_MIN_ATTEMPTS` (default 3), and `OOK_LINKCHECK_RECHECK_INTERVALS` (default 1h, 4h, 24h, 48h).
