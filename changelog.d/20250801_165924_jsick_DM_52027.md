### Backwards-incompatible changes

- A database migration is required to add a new `search_vector` column to the `authors` table. This column is used for full-text search capabilities and will be populated with a computed value based on the `given_name` and `surname` fields. Alembic migration `1ad667eab84e` to `c03d146610d8`.

### New features

- The `/authors` endpoint now supports a `search` query parameter that allows for flexible and typo-tolerant searching of authors by name. The search system automatically detects and handles various name formats:

  - "Last, First"
  - "Last, Initial"
  - "First Last"
  - Family name only
  - Given name only
  - Compound family names
  - Names with suffixes
